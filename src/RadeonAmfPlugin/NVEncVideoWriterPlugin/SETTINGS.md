# 設定項目の説明

YMM4 の「Radeon AMF プラグイン出力」を選択したときに表示される設定項目の詳細です。

---

## コーデック

| 選択肢 | 内部値 | 説明 |
| :--- | :--- | :--- |
| H.264 (AMF) | `H264` | 互換性が高く、古い機器・サービスでも再生しやすい。ファイルサイズは H.265 より大きくなる傾向がある。 |
| H.265 / HEVC (AMF) | `H265` | H.264 より圧縮率が高く、同じ画質でファイルサイズを小さく抑えられる。再生環境がH.265対応している場合に推奨。 |

**デフォルト**: H.264 (AMF)

---

## ビットレート方式

| 選択肢 | 内部値 | 説明 |
| :--- | :--- | :--- |
| 固定 (CBR) | `Fixed` | 「ビットレート（kbps）」で指定した値に固定してエンコードする。帯域が一定で安定するため、配信環境に向く。 |
| 可変 (AMF VBR) | `Variable` | シーンの複雑さに応じてビットレートを変動させる。指定ビットレートを目標値とし、最大は約1.2倍まで上振れする。静止シーンでファイルサイズを抑えやすい。 |
| 自動 (YouTube 推奨) | `YouTubeRecommended` | 「ビットレート（kbps）」欄の入力を無視し、解像度と FPS から YouTube の推奨ビットレートを自動計算して適用する（下表参照）。「ビットレート（kbps）」の設定は無効になる。 |

**デフォルト**: 自動 (YouTube 推奨)

### 「自動 (YouTube 推奨)」の計算値

| 解像度 | 30fps 以下 | 48fps 以上 |
| :--- | ---: | ---: |
| 4K (2160p) | 40,000 kbps | 60,000 kbps |
| 1440p | 16,000 kbps | 24,000 kbps |
| 1080p | 8,000 kbps | 12,000 kbps |
| 720p | 5,000 kbps | 7,500 kbps |
| それ以下 | 3,000 kbps | 4,000 kbps |

> [!NOTE]
> 可変 (VBR) および 自動 (YouTube 推奨) を選んだ場合、エンコーダへ渡す最大ビットレートは目標ビットレートの約1.2倍（上限 300,000 kbps）に設定されます。

---

## 出力品質

AMF エンコーダのプリセット（エンコード速度と画質のトレードオフ）です。

| 選択肢 | 内部値 | 説明 |
| :--- | :--- | :--- |
| 高速 | `Speed` | エンコード速度を最優先する。PreAnalysis と GPU キュー深度の変更も自動で行われる（PreAnalysis 無効・キュー深度 32 にリセット）。 |
| 標準 | `Balanced` | 速度と画質のバランスをとる。 |
| 高品質 | `Quality` | 画質を優先する。PreAnalysis が自動で有効になる。 |

**デフォルト**: 高速

> [!TIP]
> YouTube 投稿を主な目的とする場合は「標準」が速度と画質のバランスが良い選択肢です。
> アーカイブ用途やマスター素材として保存する場合は「高品質」を検討してください。

---

## ビットレート（kbps）

エンコード時の目標ビットレートを kbps 単位で指定します。

- **有効範囲**: 100 〜 200,000 kbps
- **ビットレート方式が「自動 (YouTube 推奨)」のときはこの設定は無視されます。**

**デフォルト**: 12,000 kbps（12 Mbps）

---

## GPU キュー深度

AMF エンコーダの内部入力キューに積むフレームの最大数です。この値が大きいほど YMM4 のフレーム供給とエンコーダの処理を深くパイプライン化でき、スループットが上がりやすくなります。一方、大きくしすぎると VRAM を圧迫します。

- **有効範囲**: 4 〜 32
- 「出力品質: 高速」を選ぶと自動で 32 にリセットされます。

**デフォルト**: 32

> [!TIP]
> RX 6800 XT + Ryzen 9 5900X 環境では **16〜32** が安定した結果になっています。
> VRAM 容量が少ない GPU では小さい値（8〜16 程度）から始めてみてください。

---

## PreAnalysis / 高品質解析を使う

エンコード前に AMF が各フレームの複雑さを事前解析し、ビットレートの配分を最適化するモードです。

- 有効にすると画質（特に動きの速いシーン）が向上しますが、エンコード速度はやや低下します。
- 「出力品質: 高速」を選ぶと自動で **無効** になります。
- 「出力品質: 高品質」を選ぶと自動で **有効** になります。

**デフォルト**: 無効

---

## デバッグログを書き出す

有効にすると、出力先ファイルと同じフォルダに `.amf_log.txt` が生成されます。

エンコードに失敗したときや、速度・品質に問題があると感じたときに有効にしてログを確認してください。

**デフォルト**: 無効

---

## 制約・注意事項

- **解像度は縦・横ともに偶数ピクセルでなければなりません。** 奇数サイズを指定するとエンコード開始時にエラーになります。
- **AMD Radeon GPU と AMD Software: Adrenalin Edition ドライバが必須です。** NVIDIA や Intel の GPU では動作しません。
- 出力ファイル形式は `.mp4` のみ対応しています。

---

## 推奨設定（RX 6800 XT の場合）

| 項目 | 推奨値 |
| :--- | :--- |
| コーデック | H.265 / HEVC |
| ビットレート方式 | 自動 (YouTube 推奨) または 可変 |
| 出力品質 | 標準（高品質が必要なら「高品質」） |
| GPU キュー深度 | 16〜32 |
| PreAnalysis | 高品質目的なら有効、速度優先なら無効 |
